## 데이터 소스 기준

## 1. GNB (공통 네비게이션)

GNB는 모든 페이지 상단에 고정(`fixed top-0`). 높이 `h-14`. 풀와이드 레이아웃 (`w-full px-10`).

### 1-1. 탭 이동

| 시나리오 | 전제 | 행동 | 기대 결과 |
|---------|------|------|----------|
| 홈 탭 클릭 | 현재 `/recommend` | [홈] 클릭 | `/`로 이동, [홈] 탭이 흰 배경 pill로 활성화 |
| 스마트 추천 탭 클릭 | 현재 `/` | [스마트 추천] 클릭 | `/recommend`로 이동, [스마트 추천] 탭 활성화 |
| 이미 활성 탭 클릭 | 현재 `/` | [홈] 클릭 | 페이지 재로드 없이 유지 |
| 로고 클릭 | 어느 페이지든 | DXVOD 로고 클릭 | `/`로 이동 |

### 1-2. 검색 (API 연동)

검색은 `GET /vod/search?q={query}` API를 호출하며, **300ms 디바운스** 적용.

| 시나리오 | 전제 | 행동 | 기대 결과 |
|---------|------|------|----------|
| 검색창 열기 | 검색 닫힘 | 🔍 아이콘 클릭 | 아이콘 왼쪽에 입력창(`w-52`) 등장, 자동 포커스 |
| 검색어 입력 | 검색창 열림 | "범죄" 입력 | 300ms 디바운스 후 API 호출, 드롭다운에 포스터 썸네일 + 제목 + 장르·CT_CL 표시 (최대 8건) |
| 결과 없음 | 검색창 열림 | "zzzzz" 입력 | 드롭다운: `"zzzzz" 결과 없음` 메시지 |
| 검색 결과 클릭 | 드롭다운 표시 중 | 특정 VOD 클릭 | `/series/{series_nm}` 으로 이동, 검색창 닫힘 |
| X 버튼 클릭 | 검색어 입력됨 | 입력창 내 X 클릭 | 입력값만 초기화, 검색창 유지 |
| 검색 닫기 | 검색창 열림 | 🔍 아이콘 재클릭 또는 Esc | 검색창 닫힘, 입력값 초기화 |
| 배경 딤 클릭 | 검색창 열림 | 검색창 바깥 클릭 | 검색창 닫힘 |

### 1-3. 알림 (API 연동)

알림은 `GET /user/me/notifications` API로 로드. 읽음 처리는 `PATCH /user/me/notifications/{id}/read`, 전체 읽음은 `POST /user/me/notifications/read-all`, 삭제는 `DELETE /user/me/notifications/{id}`.

| 시나리오 | 전제 | 행동 | 기대 결과 |
|---------|------|------|----------|
| 알림 열기 | 알림 닫힘 | 🔔 아이콘 클릭 | 알림 드롭다운(`w-80`) 표시, API에서 받은 알림 목록 표시 |
| 배지 카운트 | 읽지 않은 알림 있음 | - | 🔔 우상단에 빨간 뱃지 (unread_count) 표시 |
| X 클릭 (알림 개별) | 드롭다운 열림 | 알림 항목 X 버튼 클릭 | `DELETE` API 호출 → 목록에서 즉시 제거, unread_count 감소 |
| 알림 없음 | 알림 0건 | 🔔 클릭 | `알림이 없습니다` 메시지 |
| 알림 닫기 | 드롭다운 열림 | 바깥 클릭 | 드롭다운 닫힘 |
| 검색 열려있을 때 알림 클릭 | 검색창 열림 | 🔔 클릭 | 검색창 자동 닫힘, 알림 드롭다운 열림 |
| 알림 이미지 | 이미지 있는 알림 | - | `image_url` 있으면 썸네일(`w-24 h-16`) 표시, 없으면 기본 아이콘 |
| 미읽음 스타일 | `read: false` 알림 존재 | - | 미읽음 알림은 배경색·폰트 굵기로 시각 구분 |
| 개별 읽음 처리 | 미읽음 알림 존재 | 알림 항목 클릭 | `PATCH /user/me/notifications/{id}/read` 호출 → 읽음 스타일로 전환, `unread_count` 감소 |
| 전체 읽음 | 미읽음 알림 2건 이상 | [전체 읽음] 클릭 | `POST /user/me/notifications/read-all` 호출 → 모든 알림 읽음 처리, 뱃지 제거 |

### 1-4. 마이페이지 이동

| 시나리오 | 전제 | 행동 | 기대 결과 |
|---------|------|------|----------|
| 마이페이지 이동 | 어느 페이지든 | 👤 아이콘 클릭 | `/my`로 이동 |

## 2. 홈 페이지

**URL:** `/`
**데이터 소스:** `getBanner()`, `getSections()`, `getPersonalSections(userId)`, `getWatching()`
**구성:** 히어로 배너 → 이어보기 → 장르별 인기 섹션 → 개인화 추천 섹션

### 2-1. 데이터 로딩

| 시나리오 | 전제 | 행동 | 기대 결과 |
|---------|------|------|----------|
| 초기 로딩 | 페이지 진입 | - | `로딩 중...` 표시, 4개 API 병렬 호출 (`Promise.allSettled`) |
| 로딩 완료 | API 응답 수신 | - | 배너 + 이어보기 + 장르 섹션 + 개인화 섹션 순서로 렌더링 |
| 부분 실패 | 일부 API 실패 | - | 실패한 섹션만 미표시, 나머지 정상 렌더링 (`allSettled`) |
| 비로그인 상태 | `user_id` 없음 | - | 개인화 섹션 미표시, 나머지 정상 |

### 2-2. 히어로 배너

데이터 소스: `GET /home/banner` → `items[]` (로그인 여부 무관, **popular 5건 고정**)

| 시나리오 | 전제 | 행동 | 기대 결과 |
|---------|------|------|----------|
| 자동 슬라이드 | 배너 표시 | 4초 경과 | 다음 VOD로 자동 전환 (순환) |
| 인디케이터 클릭 | 배너 표시 | 특정 dot 클릭 | 해당 VOD로 즉시 전환 |
| 배너 전체 클릭 | 배너 표시 | 배너 영역 클릭 | `/series/{series_id}`로 이동 |
| 메타 표시 | - | - | CT_CL 태그(파란색) + 제목(`text-5xl`) + 장르·등급·러닝타임 + 줄거리(2줄) |
| 페이지 카운터 | - | - | 우하단에 `N / 전체` 표시 |
| 배경 이미지 | backdrop_url 있음 | - | `backdrop_url` 가로형 이미지를 배경으로 표시 |
| 배경 이미지 fallback | backdrop_url 없음, poster_url 있음 | - | `poster_url` 이미지를 배경으로 표시 |
| 이미지 없음 | backdrop_url·poster_url 모두 없음 | - | 제목 기반 그라데이션 fallback 배경 |

### 2-3. 이어보기 섹션

데이터 소스: `GET /user/me/watching` → `items[]`

| 시나리오 | 전제 | 행동 | 기대 결과 |
|---------|------|------|----------|
| 섹션 표시 | items 1건 이상 | - | 이어보기 섹션 렌더링 |
| 섹션 미표시 | items 0건 | - | 이어보기 섹션 자체가 렌더링되지 않음 |
| 카드 클릭 | 이어보기 카드 표시 | WatchingCard 클릭 | `/series/{series_nm}?episode={episode_title}`로 이동 |
| 진행바 표시 | - | - | 카드 하단에 `completion_rate`% 파란 진행바 + 퍼센트 텍스트 |
| 가로 스크롤 | 아이템 많음 | 마우스 hover 후 화살표 클릭 | 400px 단위 스크롤 |
| 화살표 표시 | - | 섹션 hover | opacity-0 → opacity-100 전환 |

### 2-4. 장르별 인기 섹션 (HorizontalSection)

데이터 소스: `GET /home/sections` → `sections[]` (CT_CL별 `인기 {ct_cl}`)

| 시나리오 | 전제 | 행동 | 기대 결과 |
|---------|------|------|----------|
| 포스터 카드 클릭 | 섹션 표시 | PosterCard 클릭 | `/series/{series_id}`로 이동 |
| 카드 hover | - | 마우스 올림 | `scale-105` 확대 + `brightness-110` (제목은 항상 카드 하단에 표시) |
| 가로 스크롤 | - | 섹션 hover + 화살표 클릭 | 400px 스크롤 |
| 장르 텍스트 | - | - | 카드 하단에 장르 텍스트 표시 |

### 2-5. 개인화 추천 섹션

데이터 소스: `GET /home/sections/{user_id}` → `sections[]`

개인화 섹션은 2종류로 분기됨:

**TOP10 섹션** (vods 중 `rank` 필드가 있는 경우):

| 시나리오 | 전제 | 행동 | 기대 결과 |
|---------|------|------|----------|
| 랭크 뱃지 | rank 값 존재 | - | 포스터 좌상단에 노란 원형 뱃지(`bg-yellow-500`)로 순위 표시 |
| 추천 문장 | rec_sentence 존재 | - | 카드 하단에 추천 사유 텍스트 표시 |
| 카드 클릭 | - | PosterCard 클릭 | `/series/{series_id}`로 이동 |

**일반 개인화 섹션** (rank 없는 경우):

| 시나리오 | 전제 | 행동 | 기대 결과 |
|---------|------|------|----------|
| 시청 비중 표시 | view_ratio > 0 | - | 섹션 하단에 `시청 비중 N%` 회색 텍스트 표시 |
| 포스터 카드 | - | 클릭 | HorizontalSection과 동일 |

**Cold Start 분기:**

| 시나리오 | 전제 | 행동 | 기대 결과 |
|---------|------|------|----------|
| 신규 사용자 (Cold Start) | 시청 이력 부족 | - | 인기 콘텐츠 기반 fallback 섹션 표시 |
| 기존 사용자 | 충분한 시청 이력 | - | 장르 선호도·벡터 유사도 기반 개인화 섹션 표시 |

## 3. 스마트 추천 페이지

**URL:** `/recommend`
**데이터 소스:** `getRecommend(userId)` → `{ source, top_vod[], patterns[] }`

### 3-1. 데이터 로딩

| 시나리오 | 전제 | 행동 | 기대 결과 |
|---------|------|------|----------|
| 초기 로딩 | 페이지 진입 | - | `로딩 중...` 표시, `GET /recommend/{user_id}` 호출 |
| 비로그인 | user_id 없음 | - | 로딩 종료, 빈 페이지 |

### 3-2. 메인 배너 캐러셀 (top_vod 배열, Cold Start 분기)

`top_vod`는 단일 객체에서 **배열(최대 5건)** 로 변경됨. 4초 자동 전환 캐러셀로 표시.

| 시나리오 | 전제 | 행동 | 기대 결과 |
|---------|------|------|----------|
| 자동 슬라이드 | top_vod 2건 이상 | 4초 경과 | 다음 VOD로 자동 전환 (순환) |
| 인디케이터 클릭 | top_vod 2건 이상 | 특정 dot 클릭 | 해당 VOD로 즉시 전환 |
| 개인화 추천 배너 | source = `personalized` | - | 파란 태그 `오늘의 TOP 추천 N` + VOD 제목 표시 |
| Cold Start 배너 | source = `popular_fallback` | - | 주황 태그 `지금 인기 있는 콘텐츠` + 인기 VOD 제목 표시 |
| 배너 클릭 | 배너 표시 | 배너 영역 클릭 | `/series/{series_id}`로 이동 |
| 배경 이미지 | backdrop_url 있음 | - | `backdrop_url` 가로형 이미지를 배경으로 표시 |
| 배경 이미지 fallback | backdrop_url 없음, poster_url 있음 | - | `poster_url` 이미지를 배경으로 표시 |
| 이미지 없음 | backdrop_url·poster_url 모두 없음 | - | fallback 그라데이션 배경 |

### 3-3. 패턴 섹션

| 시나리오 | 전제 | 행동 | 기대 결과 |
|---------|------|------|----------|
| 패턴 타이틀 | 패턴 데이터 로드 | - | `pattern_reason` 텍스트가 섹션 제목으로 표시 (예: "봉준호 감독 작품을 즐겨 보셨어요") |
| 패턴 VOD 목록 | - | - | 각 패턴별 vod_list를 PosterCard 가로 스크롤로 표시 |
| 포스터 클릭 | - | PosterCard 클릭 | `/series/{series_id}`로 이동 |
| 가로 스크롤 | - | 화살표 hover + 클릭 | 400px 스크롤 |

## 4. 시리즈 상세 페이지

**URL:** `/series/{series_id}` (series_id = `series_nm` URL 인코딩)
**데이터 소스:** `getEpisodes()`, `getProgress()`, `getPurchaseCheck()`, `getSimilar()`, `getVODDetail()`

### 4-1. 데이터 로딩

| 시나리오 | 전제 | 행동 | 기대 결과 |
|---------|------|------|----------|
| 초기 로딩 | 페이지 진입 | - | `로딩 중...` 표시, 3개 API 병렬 호출 |
| 구매 판단 | API 응답 | - | `purchaseCheck.purchased === true` OR 시청 기록 있음 OR 이어보기 쿼리파라미터 → 구매 완료 상태 |

### 4-2. 히어로 영역 (포스터 배너 / YouTube 플레이어)

| 시나리오 | 전제 | 행동 | 기대 결과 |
|---------|------|------|----------|
| 미구매 상태 진입 | purchased = false | - | 포스터 배너 + 잠금 🔒 아이콘 표시 |
| 구매 완료 상태 진입 | purchased = true | - | 포스터 배너 + ▶ 재생 버튼 오버레이 |
| 재생 버튼 클릭 | 구매 완료 | ▶ 클릭 | 에피소드 `asset_id`로 `GET /vod/{asset_id}` 호출 → `youtube_url` 추출 → YouTube IFrame 플레이어로 전환 (autoplay) |
| YouTube 재생 중 | 플레이어 활성 | - | 히어로 영역에 YouTube iframe 표시 + 우상단에 에피소드명 뱃지 |
| 재생 에러 | youtube_url 없거나 실패 | - | `영상을 재생할 수 없습니다` 메시지, 포스터로 복귀 |
| 뒤로 가기 | 어떤 상태든 | ← 버튼 클릭 | `router.back()` — 이전 페이지로 이동 |

### 4-3. 진행률 heartbeat

| 시나리오 | 전제 | 행동 | 기대 결과 |
|---------|------|------|----------|
| 재생 중 heartbeat | YouTube 재생 상태 | 30초마다 | `POST /series/{series_nm}/episodes/{episode}/progress` 호출 (completion_rate) |
| 로컬 즉시 반영 | heartbeat 전송 | - | API 재조회 없이 에피소드 목록 진행바 즉시 업데이트 |
| 재생 종료 | 에피소드 끝까지 재생 | - | completion_rate 100% 전송 + 로컬 반영 |
| playback_update | YouTube 재생 중 | 0.5초마다 | WebSocket으로 `playback_update` 메시지 전송 (광고 트리거용) |

### 4-4. 찜하기 (API 연동)

## 8. 크로스 페이지 연동 시나리오

사용자가 여러 페이지를 이동하며 상태가 연동되어야 하는 핵심 시나리오.

### 시나리오 A — 신규 콘텐츠 구매 전체 흐름

```
홈 페이지
  → 포스터 카드 클릭
  → 시리즈 상세 (미구매: 🔒 아이콘, [구매하기] 버튼)

  → 구매 페이지 오버레이 (API로 옵션 조회 → 옵션 선택 → [결제하기])
  → POST /purchases → 구매 완료 메시지 → 1.5초 후 시리즈 상세로 자동 이동
  → 시리즈 상세 (구매됨: ▶ 재생 버튼, [1화 시청하기])
```

### 시나리오 B — 이어보기 흐름

```

  → /series/{series_nm}?episode={episode_title} 이동
  → 시리즈 상세 (해당 에피소드 파란 하이라이트)
  → [▶ 이어보기] 클릭 또는 에피소드 직접 클릭
  → asset_id로 VOD 상세 조회 → YouTube 플레이어 저장된 진행률 지점부터 자동 재생 (예: 32% → 32% 지점에서 시작)
```
